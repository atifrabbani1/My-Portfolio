Flight Plan

The goal of the project was to create a website that will find the cheapest flight from the user's origin, to their destination, as well as finding events in the area of their destination for the duration of their trip.

The SkyScanner API is the API that was used to determine the cheapest flight from the origin to the destination, with the departing flight being set on the user provided start date, and the return flight being set on the user provided end date. Once the submit button is selected, an additional API call is then made to Eventbrite to retrieve events taking place during the duration of the user's trip, in the destination city that they provided. 50 events are returned containing a varitety of categories, if the user wishes to see more events, they can click the "Check For More Events" button. This will perform an additional API call to Eventbrite for the next 50 events.

Technologies Used:

  • Materialize for CSS
  • Tablesorter.js
  • Jquery
  • Google Icons and Fonts api
  • SkyScanner API
  • Eventbrite API
  • Heroku CORS-proxy

LIRI Bot

This Liri app is created to take command from users and produce the result. The main purpose of this app to show the information related to any movie, song and concert.

OVERVIEW: We have used node-spotify-api file to fetch the song detail requested by user. We have created a separate key.js file to export spotify ID and spotify secret code to access the spotify api. Then we have .env file where we have stored the SPOTIFY_ID and SPOTIFY_SECRET values. key.js is linked to .env file and later key.js file is being used to pull the ID and Secret by liri.js to fetch the song details result. Then we have used OMDB API to fetch the details of movies requested by users. We have also used Bands in Town API to fetch the details of concert with the respect to the artist name. We have created separate functions for spotify, OMDB and Bands in town to call when correct parameters are given in git bash. We have also included the command do-what-it-says, that will read the random.txt file and will run the correct command with respective to the text inside the file.

HOW TO USE: Open the liri-node-app folder in git bash. To access the liri.js file type 'node liri' in git bash and then the command (spotify-this-song, movie-this, concert-this or do-what-it-says) For example: Type 'node liri spotify-this-song Despacito' and it will pull up all the setails of the song Despacito. Song Name, Artist Name, Album Name and a preview song link.

WORKING PROJECT: LIRI Bot.PNG file is included in the github repo folder.

TECHNOLOGIES USED: Node JS, AXIOS, API, JSON.

MY ROLE: This app is solely developed by me from start to end.

Constructor Word Guess

This is a Word Guess node JS app which will take the user input as a guess word and will output the result if the word has been guessed correctly or not.

OVERVIEW: We have used Node JS Constructor for this app. The letter.js file takes the letter of the word to check if the correct letter is selected or not and will return true and false as a result. There is one more methos in this app that will compare the user input with the letters of the word and will return the word if the correct guess is selected else a underscore will be returned. We have word.js file that require letter.js file. When the Word constructor recieves a word as an argument it splits the string into character array. When user inputs the letter it runs the for loop to the length of character array and call the letter,js guess function to check for correct letter and recieves the result. And as this recieves the result, it concatenates the result to be shown on the screen. We have index.js file, this is the main file where the program initiates. This requires word.js file. It also require inquirer npm to prompt user for input and colors npm to show Correct answer selected in green and Wrong answer selected in red. As the program starts the user is asked for input. That input is sent to word .js to check if this is correct or not and recieves the resulting output on screen. If the guess limit of 10 is exceeded or correct word is guessed by user, new word is choosen to guess.

HOW TO USE: Open the Constructor-Word-Guess app in Gitbash. To access the index.js file type node index.js and hit enter. It will ask you to guess a word. Press any letter and it will return with the remaining letter to guess and will also show if the guess was correct or wrong. Once you have guessed the complete word it will generate a new random word and will ask you to guess.

WORKING PROJECT: Constructor Word Guess.PNG file is included in the github repo folder.

TECHNOLOGIES USED: Node JS, Constructor, NPM.

MY ROLE: This app is solely developed by me from start to end.

NOTE: This is a CLI App so it cannot be deployed to Github Pages. So no Website for this app.

Mini Amazon

This is a Mini Amazon CLI app. Where Customers can use it for shopping different items online. Managers can manage the inventory and keep an eye on the inventory item available. Supervisors can manage the Financial part of the business.

OVERVIEW: We have used Node JS and SQL to develop this app. Bamazon.sql file describes the schema of the app. BamazonCustomer.js file shows the user opptions to shop from the available items and to calculate the final prcie after the shopping is completed. BamazonManager.js file is for the managers who keep an eye on inventory, they can add more items in inventory and also add new items to the list. BamazonSupervisor.js file is used by the supervisor where they can view the financial details of every department and also add new department to the store.

HOW TO USE: Open the Mini-Amazon app in Gitbash. To access the Customer part type node bamazonCustomer.js and it will ask for the item id you want to buy and the quantity and will return the total amount to pay. To access the Manager part type node bamazonManager.js and it will give you options to view inventory, add more items to inventory, add new items to inventory and see if the particular item is less than 5. To access the Supervisor part type node bamazonSupervisor.js and it will give him option to view the finances of every department and to add new department.
